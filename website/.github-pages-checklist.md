# GitHub Pages Deployment Checklist

Use this checklist to ensure successful deployment to GitHub Pages.

## Pre-Deployment Checklist

### Repository Configuration
- [ ] Repository is public (or GitHub Pages is enabled for private repo with GitHub Pro/Team)
- [ ] GitHub Pages is enabled in Settings > Pages
- [ ] Source is set to "GitHub Actions" (not branch)

### Code Configuration
- [ ] `website/docusaurus.config.js` has correct settings:
  - [ ] `url: 'https://lemniscate-world.github.io'`
  - [ ] `baseUrl: '/Neural/'`
  - [ ] `organizationName: 'Lemniscate-world'`
  - [ ] `projectName: 'Neural'`
  - [ ] `trailingSlash: false`

### Required Files
- [ ] `.github/workflows/deploy-docs.yml` exists
- [ ] `website/static/.nojekyll` exists (prevents Jekyll processing)
- [ ] `website/package.json` exists with build script
- [ ] `website/package-lock.json` exists (run `npm install` if missing)

### Local Testing
- [ ] Test build locally: `cd website && npm install && npm run build`
- [ ] No build errors or warnings
- [ ] Preview locally: `npm run serve` and check http://localhost:3000

## Deployment Checklist

### Initial Deployment
- [ ] Push changes to `main` branch
- [ ] Or manually trigger workflow via Actions tab
- [ ] Monitor workflow in Actions tab
- [ ] Workflow completes successfully (green checkmark)
- [ ] Wait 1-2 minutes for deployment to propagate

### Verification
- [ ] Visit https://lemniscate-world.github.io/Neural/
- [ ] Homepage loads correctly
- [ ] CSS and images load properly
- [ ] Navigation links work
- [ ] Documentation pages are accessible
- [ ] No console errors in browser DevTools

### Post-Deployment
- [ ] Test on mobile devices
- [ ] Check all major pages:
  - [ ] Homepage (/)
  - [ ] Docs (/docs/intro)
  - [ ] Playground (/playground)
  - [ ] Pricing (/pricing)
  - [ ] Showcase (/showcase)
  - [ ] Comparison (/comparison)

## Troubleshooting Checklist

### If Build Fails
- [ ] Check Actions tab for error logs
- [ ] Verify Node version (should be 18+)
- [ ] Check for syntax errors in JS/JSX files
- [ ] Ensure all dependencies are in package.json
- [ ] Run `npm install` locally and commit package-lock.json

### If Site Shows 404
- [ ] Verify GitHub Pages is enabled
- [ ] Check source is "GitHub Actions" not "Branch"
- [ ] Wait 2-5 minutes after deployment
- [ ] Clear browser cache and retry
- [ ] Check workflow completed successfully

### If Assets Don't Load
- [ ] Verify `baseUrl: '/Neural/'` is set correctly
- [ ] Check browser console for 404 errors
- [ ] Ensure static files are in `website/static/`
- [ ] Rebuild and redeploy

### If Links Don't Work
- [ ] Use relative links in markdown (e.g., `./intro` not `/intro`)
- [ ] For external links, use full URLs
- [ ] Check `trailingSlash: false` is set
- [ ] Verify internal links in navbar/footer

## Performance Checklist

- [ ] Images are optimized (compressed)
- [ ] Large files (>1MB) are necessary
- [ ] Bundle size is reasonable (<5MB)
- [ ] Page load time is acceptable (<3s)

## SEO Checklist

- [ ] Meta tags are configured in docusaurus.config.js
- [ ] Sitemap.xml is generated
- [ ] robots.txt is present (if needed)
- [ ] Google Analytics is configured (optional)
- [ ] Social share image exists

## Maintenance Checklist

### Regular Updates
- [ ] Update dependencies regularly: `npm update`
- [ ] Test updates locally before deploying
- [ ] Monitor GitHub Actions for workflow failures
- [ ] Check site after each deployment

### Content Updates
- [ ] Follow content guidelines in README.md
- [ ] Test markdown rendering locally
- [ ] Check for broken links before pushing
- [ ] Update last modified dates

## Emergency Rollback

If deployment breaks production:

1. [ ] Go to Actions tab
2. [ ] Find last successful workflow run
3. [ ] Click "Re-run jobs"
4. [ ] Or revert the commit: `git revert HEAD && git push`
5. [ ] Monitor new deployment

## Custom Domain Setup (Optional)

If using custom domain:

- [ ] Create `website/static/CNAME` with domain name
- [ ] Configure DNS A/CNAME records
- [ ] Update `url` in docusaurus.config.js
- [ ] Change `baseUrl` to '/' 
- [ ] Add custom domain in Settings > Pages
- [ ] Enable "Enforce HTTPS"
- [ ] Wait for SSL certificate (up to 24 hours)

## Notes

- Deployments typically take 2-5 minutes
- GitHub Pages has 1GB size limit
- Free tier has 100GB bandwidth/month
- HTTPS is automatic and enforced
- CDN is automatically configured
