## Image Guidelines

### Screenshots

- **Resolution**: 1920x1080 minimum
- **Format**: PNG (lossless)
- **DPI**: 96 (web) or 300 (print)
- **Compression**: Use tools like TinyPNG
- **Naming**: lowercase-with-hyphens.png

### Diagrams

- **Format**: PNG or SVG (vector preferred)
- **Colors**: Match Aquarium dark theme
- **Fonts**: Sans-serif, readable
- **Labels**: Clear and concise

### GIFs

- **Resolution**: 1280x720 recommended
- **Frame rate**: 15-30 fps
- **Duration**: 3-10 seconds loops
- **File size**: < 5MB if possible
- **Compression**: Use tools like Gifsicle

## Creating Screenshots

### Tools

**Windows**:
- Snipping Tool (Win+Shift+S)
- ShareX (advanced)
- Greenshot (open source)

**macOS**:
- Command+Shift+4 (area)
- Command+Shift+5 (advanced)
- CleanShot X (paid)

**Linux**:
- Flameshot
- Spectacle (KDE)
- GNOME Screenshot

### Best Practices

1. **Clean browser**: No bookmarks bar, clear address
2. **Consistent window size**: 1920x1080 or similar
3. **Hide sensitive info**: No personal data
4. **Show relevant content**: Focus on feature
5. **Use cursor**: Point to important elements
6. **Annotate**: Add arrows, highlights if needed

## Creating GIFs

### Tools

- **OBS Studio** (screen recording)
- **ScreenToGif** (Windows)
- **Kap** (macOS)
- **Peek** (Linux)

### Process

1. Record screen at 1280x720
2. Keep under 10 seconds
3. Trim unnecessary frames
4. Optimize with ezgif.com or Gifsicle
5. Test on documentation page

## Image References in Documentation

Use relative paths:

```markdown
![Alt text](../images/aquarium/image-name.png)
```

### Alt Text Guidelines

- **Be descriptive**: "DSL editor with example code"
- **Not just filename**: Not "screenshot-1.png"
- **Accessibility**: Helps screen readers
- **SEO**: Helps search engines
